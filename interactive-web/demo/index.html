How It Works

The 4-step analysis pipeline

Click any step to see what happens under the hood.

01 / normalize
Ingest
Accept text, URL, or file upload
Normalizes input from any source (pasted text, URL via web_fetch, uploaded file) into a clean Markdown document. Strips navigation chrome, ads, and formatting artifacts. Produces normalized.md.
02 / extract
Analyze
Classify type & extract structure
Identifies article type (Tutorial, Opinion, Data-driven, etc.), dominant structure (Linear, Timeline, Comparison…), target reader, and 3–7 key concepts. Produces structure.json.
03 / blueprint
Plan
Select model & design direction
Applies degradation logic (does this content genuinely benefit from interactivity?), selects an interaction model from 10 patterns, and commits to one of 4 aesthetic directions. Produces blueprint.json — the implementation contract.
04 / build
Generate
Produce single-file HTML artifact
Builds a fully self-contained HTML file with CSS custom properties, responsive layout, staggered animations, keyboard accessibility, and at least one interaction that materially reduces cognitive load. Never copies paragraphs verbatim.

Installation

Add to your Claude Code setup

Three ways to install the skill:

# In Claude Code, run:
/skills install zerohzz/zz-skills/interactive-web

# Then use it:
# Give Claude any article and say:
# "Turn this into an interactive webpage"
# Download and place in your skills directory:
~/.claude/skills/interactive-web/

# Required files:
# SKILL.md
# references/design-principles.md
# references/visualization-patterns.md
# references/content-extraction-guide.md
# assets/component-patterns.md
# scripts/normalize_article.py
# scripts/extract_structure.py
# scripts/build_page_plan.py
git clone https://github.com/zerohzz/zz-skills.git
cd zz-skills/interactive-web

# Copy skill/ directory to your Claude skills folder:
cp -r skill ~/.claude/skills/interactive-web
